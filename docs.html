Quick start

  1. Open Gledhill Metadata and choose Pick Photos, or drag image files into the main window.
  2. Select the photos you want to update from the photo browser.
  3. Choose a metadata template and naming convention.
  4. Review the rename preview, then click Apply.
  5. Confirm the Apply warning so the app can update metadata and filenames.
Tip: Gledhill Metadata works locally on your computer. Your images are not uploaded to a cloud service.

Linux install scripts

You can install and uninstall the Linux AppImage directly from curl:

curl -fsSL https://raw.githubusercontent.com/Nerd-or-Geek/Gledhill-Metadata/main/linux/install.sh | bash

Photo browser

Use the photo browser to review loaded images, choose which files are selected, and inspect metadata for the focused photo.

List view

Shows a compact thumbnail and filename for each image. This is useful on smaller screens or when you need to scan many filenames quickly.

Grid view

Shows larger thumbnails with filenames. Adjust the grid thumbnail size in Settings to fit more images or see bigger previews.

Metadata templates

Templates let you save reusable metadata values and apply them consistently across selected photos.

Recommended XMP field order

  1. Title
  2. Description / Subject
  3. Tags / Keywords
  4. Comments / Headline
  5. Authors / Creator
  6. Copyright

The app also supports headline fields such as IPTC.Headline and XMP.Headline.

Renaming files

Naming conventions help generate predictable file names for batches. Select a convention, then check the live rename preview before applying changes.

Safe workflow: Test with a small group of copied photos first when creating a new naming pattern.

Settings

Default selections

Choose a default metadata template and naming convention so the main screen opens with your preferred workflow ready.

Photo browser display

Switch between list and grid mode, then resize grid thumbnails to match your screen size and working style.

Keyboard shortcuts

Set hotkeys for Select All, Unselect All, Delete Selected, Apply, and Pick Photos. Defaults use Command on macOS and Ctrl on Windows/Linux.

Updates and support

Use the update button to check for new releases and the bug-details button to copy useful diagnostics for developers.

Troubleshooting

Pick Photos does not open Finder on macOS

Make sure Gledhill Metadata has permission to access files and folders in macOS System Settings. Drag and drop can still work even when picker access is blocked.

Metadata clear verification reports remaining tags

Some embedded metadata, such as color profiles, may remain because they are important for image appearance. If you need a specific tag removed, copy bug details and include the exact file type and tag name.

The app opens too small or too large

Resize the window once, then close the app normally. Gledhill Metadata saves the last window size and restores it on launch.

Bug reports

Good bug reports make fixes much faster. In Settings, click Copy Bug Details, then paste that information into a GitHub issue or support message.

  • What you expected to happen.
  • What actually happened.
  • Your operating system and app version.
  • A sample filename or metadata tag if the issue is metadata-related.
